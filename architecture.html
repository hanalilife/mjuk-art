系統分層架構圖

前端層
1

使用者

上傳客戶提供的參考圖片

2

搜尋介面 UI

拖曳圖片上傳、搜尋操作流程設計

3

Frontend

實作使用者操作（拖曳上傳、圖片顯示與互動）

API層
4

API 請求

前端將圖片送至後端（POST /search）

5

後端服務（FastAPI）

接收圖片，並串接搜尋與比對流程

AI搜尋層
6

CLIP Embedding

將圖片轉換為可計算的特徵向量

7

向量比對

比較向量相似度，找出最接近的款式

資料層
8

資料來源

圖片資料、CSV 價格、JSON 標籤

9

搜尋結果

回傳款式圖片及資訊

關鍵技術決策理由

技術 選擇原因
HTML/CSS/JS 快速建立可展示的操作介面，方便調整 UI 與互動流程
FastAPI 作為前端與搜尋流程之間的 API 橋樑，負責接收圖片與回傳結果
CLIP 可將圖片轉為向量，支援以圖片特徵進行搜尋
向量比對 用相似度排序，快速找出資料庫中最接近的款式
CSV 儲存尺寸、價格等結構化資料，方便現場人員維護
JSON tags 儲存分類與標籤關係，支援多條件搜尋
本地部署 符合內部資料使用需求，降低資料外流風險
PIL（圖片處理） 將搜尋結果圖片整理並輸出為報價用圖

未來規劃

功能升級

  • 自動標籤 AI 輔助
    自動辨識圖片內容，產生分類與標籤，減少人工建檔時間
  • 圖片特徵自動分類
    依據圖片風格與裝飾特徵，自動進行分類與整理
  • 價格推薦
    根據歷史資料，提供相似款式的建議報價範圍

架構演進

  • 雲端部署
    將系統由本地端轉為雲端服務，支援多人同時使用與遠端存取
  • 向量資料庫
    提升大量圖片比對效能，支援更快速且可擴展的相似度搜尋
  • 多資料來源整合
    串接圖片與訂單資料，提供過往案例作為報價依據

產品擴展

  • 設計模組
    提供拖曳式設計工具，讓客戶可自訂蛋糕款式
  • 自動估價
    依據設計內容與歷史資料，提供即時報價建議
  • 多語系
    支援多語言介面，提升不同市場的使用體驗
